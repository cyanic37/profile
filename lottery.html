게임 설명

이 게임은 즉석 복권을 모티브로 한 게임입니다. 복권을 긁으면 복권 기본 비용만큼 차감하고, 은박지를 긁어서 승리 조건을 만족하면 상금을 주는 형식의 게임입니다.

프로젝트 설명

역할

이 프로젝트에서 제가 맡은 역할은 서버단과, 서버와 통신하는 자바스크립트 AJAX 기능 그리고 자바스크립트 페이스북 로그인 기능 파트입니다.

Django 사용 이유

서버는 Python & Django로 구현하였습니다. Django는 스프링에 비해서 생산성이 뛰어나고, 파이썬이 루비보다 활용할 수 있는 곳(데이터 가공, TF)이 많아 공부하기 좋아 선택하게 되었습니다.

User 모델

Abstract User 모델을 확장하여 유저 DB 필드를 확장하고, Facebook login api를 활용하여 페이스북 정보로 유저를 등록하였습니다. 이후에 안드로이드 버전을 구현하면서 Google Sign-in api를 활용하여 구글 정보로도 유저를 등록할 수 있게 만들었습니다.

로그인

인증 시스템은 처음에 all-auth라는 Django 프로젝트를 활용하였으나, all-auth 프로젝트가 페이스북 게임이 구동되는 iframe을 제대로 지원하지 않아 자바스크립트 sdk로 직접 구현하는 방식으로 바꾸었습니다.

Web Server -> API Server

이 게임을 안드로이드와 iOS로 포팅하기 위해서 Django REST framework를 사용하여 REST API 형태로 서버를 개선하였습니다. 웹과 모바일 어플에서 API 접근 권한을 설정하기 위해서 Django REST framework jwt(JSON Web Token)를 사용하였습니다.

결제

웹 버전은 Facebook Web Payment api를 활용하여 결제 시스템을 구축하였고 안드로이드 버전은 유니티로 클라이언트를 구현하고 Unity iAP을 통하여 결제 시스템을 구축하였습니다.

게임 정보 통신

게임은 은박지를 클릭하였을 때 은박지 내부의 정보는 서버와 AJAX 통신을 하여 생성하고, 은박지를 모두 긁었을 때는 게임 결과를 정산하기 위해 서버와 AJAX 통신을 하도록 구현하였습니다.

로그

게임 플레이 내역과 구매 내역을 저장하기 위해서 로그 DB를 추가하였습니다. 하지만 서버를 계속 개선하면서 디버깅을 위한 로그 시스템의 필요성을 느끼고 있습니다.

배포

Nginx - Gunicorn - Django 형식으로 웹 서버를 구성하였습니다. 처음에는 공부를 목적으로 docker를 사용하였습니다. 새로운 서버에 프로젝트를 설치하는 것은 매우 편하였지만, 사용자수가 매우 적은 단일 서버를 운영하기 때문에 굳이 docker를 사용할 이유를 못 느껴 스크립트를 통해서 재배포를 하도록 바꾸었습니다. 하지만 하나의 대형 서비스를 운영하는 경우 젠킨스와 쿠버네티스 같은 툴과 같이 사용하면 정말 유용할 것 같다는 것을 느꼈습니다. 또 Nginx를 공부하면서 로드 밸런싱 개념도 익힐 수 있었습니다.
